## Commands

```bash
bun run dev           # Start dev server
bun run build         # Production build (Cloudflare Workers target)
bun run build:dev     # Debug build
bun run lint          # ESLint
bun run format        # Prettier (auto-fix)
bun run test:e2e      # Playwright E2E tests (requires E2E_PASSWORD env var)
bun run test:e2e:ui   # Playwright interactive UI mode
```

### Cloudflare Workers deployment

```bash
bun run build
wrangler deploy
# Required secrets (set once per environment):
wrangler secret put RESEND_API_KEY
wrangler secret put ADMIN_EMAIL
wrangler secret put SUPABASE_SERVICE_ROLE_KEY
```

Playwright tests live in `e2e/`. Auth setup saves session to `e2e/.auth/user.json` (git-ignored). Before first run:

```bash
export E2E_EMAIL=osmanicm@gmail.com
export E2E_PASSWORD=<your-supabase-password>
bun run test:e2e
```

## Architecture

### Stack

- **TanStack React Start** (full-stack React) + **TanStack Router** (file-based routing)
- **Vite 7** via `@lovable.dev/vite-tanstack-config` (handles Tailwind, TSX, Cloudflare, path aliases internally â€” `vite.config.ts` is minimal)
- **Supabase** (PostgreSQL + Auth) â€” client at `src/integrations/supabase/client.ts`
- **React Query** â€” server state, 30s stale time, 1 retry, wrapped per feature in `src/data/`
- **Cloudflare Workers** for deployment (`wrangler.jsonc`)

### Routing

Routes live in `src/routes/` and are file-based. `routeTree.gen.ts` is auto-generated by TanStack Router â€” never edit it manually. Each route file exports a `createFileRoute(...)` with `component` and optional `loader`.

Role-based access is enforced at the route level using the `AuthProvider` context (`src/hooks/useAuth.tsx`). Admin-only routes redirect agents to a Forbidden screen.

### Data Layer

All Supabase queries go through custom hooks in `src/data/`. Each hook uses `useQuery` / `useMutation` from React Query. Mock data for development is in `src/data/mock.ts`.

The database uses **Row-Level Security (RLS)** and a `user_roles` table (`admin` / `agent`). Always consider RLS policies when writing new queries.

### Component Structure

- `src/components/ui/` â€” shadcn/ui base components (new-york style, Radix UI primitives, Lucide icons). Configured via `components.json`.
- `src/components/layout/` â€” `AppShell`, `Sidebar`, `Topbar`, bottom nav for mobile.
- Feature components are grouped by domain: `availability/`, `events/`, `properties/`, `leads/`, `users/`.

### Styling

Tailwind CSS 4 with a custom OkLch color palette (deep green primary, gold accents). CSS variables are defined in `src/styles.css`. Path alias `@/*` maps to `src/*`.

### Patterns

- **Immutable data** â€” never mutate state directly; always return new objects.
- **Soft deletes** â€” rows use a `deleted_at` column, not hard deletes.
- **Forms** â€” React Hook Form + Zod schemas for validation.
- **Drag-and-drop** â€” `@dnd-kit` used in availability management.
- **PDF export** â€” `@react-pdf/renderer` for generating property documents.
- **Toast notifications** â€” Sonner (`src/components/ui/sonner.tsx`).

### Database Migrations

Migrations live in `supabase/migrations/`. Apply them via the Supabase CLI (`supabase db push`) or the Supabase MCP tools. 16 migrations track schema evolution from Aprilâ€“May 2026.
